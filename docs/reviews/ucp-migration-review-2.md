## Independent re-verification (Dev MCP + ground truth)

- **Idempotency (change #3).** Dev MCP Cart MCP: _"cancel_cart … Requires `meta[\"idempotency-key\"]` (UUID) in addition to `meta[\"ucp-agent\"]`."_ Dev MCP Checkout MCP: _"cancel_checkout … Requires `meta[\"idempotency-key\"]` (UUID) in addition to `meta[\"ucp-agent\"]`"_ and _"complete_checkout … Requires `meta[\"idempotency-key\"]`."_ Ground-truth `ucp-tools-list.json`: `complete_checkout` schema has `meta.required = ["ucp-agent","idempotency-key"]` (docs and schema agree); `cancel_checkout` and `cancel_cart` schemas have `meta.required = ["ucp-agent"]` only (docs-require vs schema-optional conflict). The plan's §5.4 and AL-UCP-8 describe this three-way relationship **exactly** — docs require the key on all three, schema under-specifies the two cancel tools, `complete_checkout` has no discrepancy. Neither overstated nor understated. Verified accurate.
- **Rate-limit / `-32000` (change #5).** Dev MCP: _"Transport-level failures (… rate limiting …) are returned as JSON-RPC error with code -32000, or -32001 for discovery errors … When the server rate-limits your request, retry after the delay specified by the HTTP Retry-After response header … honor Retry-After and apply exponential backoff with jitter when the header is absent."_ Confirmed: a `-32000` can arrive in a **200 body** while HTTP `Retry-After` is honored. The live `callTool` at `mcp.server.js:88` throws on `res.status === 429` before parsing the body, and at line 102 maps any `data.error` to generic `rpc_error` with no `Retry-After` read — so a `-32000` rate-limit body would be mis-mapped today. The plan's AL-UCP-13 / §6.5 / step 5 / step 9 / §10.4 all require reading `Retry-After` in BOTH paths and a dedicated `-32000`+`Retry-After` unit test. Verified accurate and complete.
- **Totals (change #7).** Dev MCP: _"Cart/checkout pricing lives in `result.totals[]`; there is no `result.cost` field"_ and the printer contract _"Render `result.totals[]` in the order provided … Do not reorder, recompute, filter, or aggregate."_ The live `normalizeCart` (`mcp-normalize.js:296`) reads `rawCart.cost?.total_amount` — confirmed broken against UCP. The plan (§6.5, AL-UCP-5, step 6) rewrites it to read `totals[]` and select `type === "total"`, with an explicit order-preserving / no-recompute instruction that matches the printer contract, and a safe fallback that is Anti-Stubbing-compliant. Verified accurate.
- **`"detail"` removal (change #4).** Confirmed in live code: `getProductDetails` imported at route lines 6-9, `normalizeProductDetail` at lines 10-14, `case 'detail':` at lines 109-124 calling both. The plan's step 7a deletes the case block AND the imports (not commented out), and steps 5/6 delete the now-dead `getProductDetails` / `normalizeProductDetail` / `normalizeProductDetailsMoney` after confirming no other caller. Verified accurate.
